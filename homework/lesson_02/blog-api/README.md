blog-api
基于 Gin + GORM 实现的标准博客后端 API，包含用户认证、文章管理、评论管理、统一响应处理与日志落盘，结构规范、可直接运行和扩展。

技术栈

- Go 版本：1.25+

- Web 框架：Gin

- ORM 框架：GORM

- 数据库：SQLite（无需额外部署，自动生成）

- 身份认证：JWT（JSON Web Token）

- 密码加密：bcrypt

- 日志处理：按天切割，存储至 logs 目录，同时输出控制台

功能清单

1. 用户模块

- 用户注册：用户名/邮箱唯一，密码 bcrypt 加密存储

- 用户登录：校验用户名密码，成功返回 JWT Token（有效期 24 小时）

2. 文章模块

- 创建文章：仅已认证用户可创建，需提供标题和内容

- 文章列表：公开可访问，查询所有文章

- 文章详情：公开可访问，根据文章 ID 查询单篇文章

- 更新文章：仅文章作者可更新自己的文章

- 删除文章：仅文章作者可删除自己的文章

- 钩子函数：创建文章后，自动增加作者的文章计数（post_count）

3. 评论模块

- 创建评论：仅已认证用户可发表，需关联具体文章

- 评论查询：公开可访问，根据文章 ID 查询该文章的所有评论

- 删除评论：已实现删除功能（可后续优化为仅评论作者可删除）

- 钩子函数：删除最后一条评论后，自动将文章评论状态改为 no_comments

4. 通用功能

- 统一响应：所有接口返回格式一致，包含 code、msg、data 字段

- 错误处理：全局异常捕获，针对不同错误返回对应 HTTP 状态码和提示信息

- 日志记录：请求日志按天切割存储，包含请求时间、方法、路径、IP、状态码、耗时

- 跨域支持：支持跨域请求，适配前端调用
